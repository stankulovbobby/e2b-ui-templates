# Mock Data Modeling Skill

Mock data should feel realistic, consistent, and useful for the UI.

Guidelines:

- Define TypeScript types for major entities.
- Keep sample counts reasonable for a demo template.
- Use realistic solar names, stages, regions, values, durations, and risks.
- Keep numbers internally consistent: pipeline value, signed revenue, and contract values should feel plausible together.
- Include both positive and risky cases so the dashboard has meaningful contrast.
- Use specific actions, not generic notes.

For executive dashboards, include data for:

- KPI cards
- pipeline stages
- revenue trend
- operations health
- at-risk projects
- team performance
- regions
- bottlenecks or insights
